git push / pull

02 — Commands Practiced

bash — git-version-control-practice
$git clone https://github.com/omoladeaboaba/git-version-control-practice.git
Cloning into 'git-version-control-practice'...
$git add .# stage all changes
$git commit -m "updated README with project details"
$git push origin main
main -> main
$git checkout -b feature/add-portfolio-page# new branch
$git push origin feature/add-portfolio-page
$git revert f88c733 --no-edit# undo a commit safely
$git fetch origin
$git merge origin/feature/add-portfolio-page
$git branch -m feature/add-portfolio-page feature/portfolio-page-v1# rename
$git push origin --delete feature/add-portfolio-page# remove old
$git log --oneline
3bb18ad Merge pull request #2
dfeff0a Revert "edited index.html - testing revert"
ab83d0b updated README with project details
e61f5b4 initial commit - added README

03 — What I Did & Why

Step 1

Created & Cloned the Repository

Set up a new public repository on GitHub, then cloned it to my local machine using the command line to begin working.

git clone [url]
Step 2

Made & Pushed Commits

Edited the README, staged changes and committed them with clear messages, then pushed to GitHub.

git add . → git commit → git push
Step 3

Created a Feature Branch

Created a new branch to work on a portfolio page independently without touching the main branch.

git checkout -b feature/add-portfolio-page
Step 4

Reverted a Commit

Intentionally made a change then used git revert to safely undo it, preserving the commit history.

git revert [commit-id] --no-edit
Step 5

Fetched, Merged & Renamed

Fetched remote changes, merged the feature branch into main, then renamed the branch and deleted the old reference.

git fetch → git merge → git branch -m
Step 6

Created & Merged a Pull Request

Opened a pull request on GitHub, reviewed the changes and merged it into the main branch.

GitHub → Pull Requests → Merge
